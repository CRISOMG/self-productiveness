RESUMEN EJECUTIVOYourfocus evoluciona de ser una herramienta de transcripción deficitaria a un Sistema de Prótesis Cognitiva financieramente sostenible. El objetivo central es transformar el monólogo interno del usuario en sistemas de ejecución (BPM Personal), utilizando la Inteligencia Artificial no como un fin, sino como un medio para alcanzar la excelencia profesional.La viabilidad del proyecto depende de un pivotaje técnico inmediato: desacoplar la transcripción (costosa en multimodal) del razonamiento (barato en texto), reduciendo el costo marginal en un 85% y permitiendo financiar las metas de vida del founder (Ingeniería Mecatrónica y Vivienda Propia) con una base de usuarios reducida (<200 usuarios activos).2. FUNDAMENTOS FILOSÓFICOS Y VISIÓNEl proyecto no es solo software; es un mecanismo de movilidad social y acondicionamiento mental.2.1. El Propósito: La Consecuencia InevitableSalir de la pobreza y alcanzar la excelencia técnica no ocurre por suerte, sino como consecuencia inevitable de sistemas robustos. Yourfocus es la herramienta que garantiza esa robustez.2.2. La Psicología del ProductoExternalización del Monólogo: Liberar la RAM biológica (cerebro) volcando el caos en audio para que el sistema lo estructure.Efecto Compuesto: Pequeñas bitácoras diarias $\rightarrow$ Base de conocimiento masiva $\rightarrow$ Seniority demostrable.Métrica de Acondicionamiento (KPI): El "Pomodoro" como unidad de medida de resistencia mental.4 Pomodoros (2h): Mantenimiento.8 Pomodoros (4h): Jornada Estándar.16 Pomodoros (8h): R&D / Alto Rendimiento.3. ARQUITECTURA TÉCNICA (EL PIVOTE)Problema Detectado: El envío de audio crudo (25 min) a modelos multimodales genera un costo de ~$25 USD/mes por usuario, haciendo inviable el precio de suscripción.Solución: Arquitectura de Orquestación Híbrida ("Pipeline Malacate").3.1. Flujo de Datos OptimizadoCaptura: El usuario graba el audio (Web/PWA).Transcripción (El Oído): Se ejecuta Whisper (modelo small o base).Opción A (Ideal): Local en el dispositivo del usuario (WASM) $\rightarrow$ Costo $0.Opción B (Servidor): Instancia barata con faster-whisper $\rightarrow$ Costo marginal bajo.Estructuración (El Cerebro): Se envía solo el texto + timestamps al LLM (Gemini Flash / GPT-4o-mini).Prompt del Sistema: "Analiza este texto, extrae: Idea Principal, Tangentes (Ideas Secundarias), Tareas y Referencias a OKRs".Persistencia: Se guarda el JSON resultante en Supabase.3.2. Restricciones de Diseño (Constraints)Micro-Bitácoras (1-3 min): Fomentar grabaciones cortas para actualización de estado ("Check-in"). Procesamiento casi instantáneo.Batch Processing (Fin del día): Si el usuario graba 25 min, se procesa en cola de baja prioridad para no saturar recursos inmediatos.4.
